##  Day 1: Introduction to Terraform & Infrastructure as Code (IaC)

###  What is Infrastructure as Code (IaC)?
Infrastructure as Code (IaC) involves writing and executing code to define, deploy, update, and destroy infrastructure in a consistent and automated way.

---

##  Categories of IaC Tools

- **Ad Hoc Scripts**
  - Examples: Bash, Python, Ruby

- **Configuration Management Tools**
  - Examples: Chef, Puppet, Ansible

- **Server Templating Tools**
  - Examples: Docker, Packer, Vagrant

- **Orchestration Tools**
  - Examples: Kubernetes, Amazon EKS, Docker Swarm, Nomad

- **Provisioning Tools**
  - Examples: Terraform, AWS CloudFormation, OpenStack Heat, Pulumi

---

## ✅ Benefits of Infrastructure as Code

- ⚡ **Speed and Safety** – Faster deployments with reduced risk of human error  
-  **Self-Service** – Fully automated infrastructure provisioning  
-  **Easy Documentation** – Code serves as documentation  
- ♻️ **Reusability** – Easily reuse configurations across projects  
-  **Version Control** – Track changes using Git or other VCS tools  

---

## ⚖️ Declarative vs Imperative Approaches

###  Imperative Approach
- Defines **how** to achieve the desired state  
- Step-by-step instructions are provided  

**Example:**
- Create a server  
- Configure networking  
- Install software  

---

###  Declarative Approach
- Defines **what** the desired final state should be  
- The tool determines **how** to achieve it  

**Example:**
- Specify that a server with certain specifications should exist  

*Declarative approaches are generally preferred because they are simpler, easier to maintain, and less error-prone.*
