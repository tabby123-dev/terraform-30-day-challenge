## вҡ–пёҸ Declarative vs Imperative Approaches

### нҙё Imperative Approach
- Defines **how** to achieve the desired state  
- Step-by-step instructions are provided  

**Example:**
- Create a server  
- Configure networking  
- Install software  

---

### нҙё Declarative Approach
- Defines **what** the desired final state should be  
- The tool determines **how** to achieve it  

**Example:**
- Specify that a server with certain specifications should exist  

нІЎ *Declarative approaches are generally preferred because they are simpler, easier to maintain, and less error-prone.*
